cmd_list_tourist

描述

列出所有登记的游客id。后台通过cmd_ret主题返回是否成功。

消息实例

{"cmd_id": "47960"}

字段

cmd_id：指令id，每次调用指令都应当使用不同的id，以便在返回时区分是哪条指令的返回值。字符串型。

返回

返回的"output"字段是一个字符串。它是一个JSON列表，列表的元素都是闸机id字符串。例：

rqst_val_acc_machine_id

rqst_val_acc_machine_id

宏

machine_id 替换为 闸机的id。

描述

该主题包含id为machine_id的闸机发出的"验证通行（validate access）"请求。要求验证某位游客是否可以通过。

消息实例

{ "tourist_id": "19475", "body_temp": 36.8 }

字段

tourist_id：游客的id。字符串型。

body_temp：游客的体温。浮点数型。

rply_val_acc_machine_id

rply_val_acc_machine_id

宏

machine_id 替换为 闸机的id。

描述

该主题包含后台对id为machine_id的闸机发出的"验证通行（validate access）"请求的回复。

消息实例

{ "tourist_id": "73169", "status": "granted", "error_code": 0 }

字段

tourist_id：游客的id。字符串型。

status：验证结果，字符串型。可能的值：

  • "granted"：允许通过；
  • "invalid_id"：拒绝通过，非法的游客id；
  • "high_temp"：拒绝通过，体温过高。

    • error_code：浮点数型。参见后台错误代码。

    cmd_add_tourist

    描述

    tourist_id：游客的id。字符串型。

    cmdret

    cmdret

    描述

    该主题包含后台执行控制端的指令后的返回值。所有指令的返回值共享此主题。

    消息实例

    { "cmd_id": "49857", "success": true, "output": "Machine 87954 added." }

    字段

    cmd_id：指令id，每次调用指令都应当使用不同的id，以便在返回时区分是哪条指令的返回值。字符串型。

    success：指令执行是否成功。布尔型。

    output：指令的输出。字符串型。

    数据传输协议

    数据传输协议

    version 3.0

    数据流向

    协议的用户有闸机、后台和控制端三种。

    控制端的任务是向后台登记和注销闸机id和用户id。后台不会响应未登记的闸机发出的请求，也不会允许未登记的游客通过闸机。有必要的话，控制端也可以订阅其他主题，以监视系统的运作情况。

    闸机的任务是取得游客的id和体温，游客的id存放在他的芯片中。闸机将这些信息发送给后台，后台判断该游客是否可以进入，并返回结果给闸机。注意，在此之前控制端必须登记此闸机id，否则后台不会响应。

    后台的任务是处理控制端和闸机的指令和请求，并一一作出回应。

    数据主要有四种流向：

    • 从闸机到后台，称为请求（rqst）
    • 从后台到闸机，称为回复（rply）
    • 从控制端到后台，称为指令（cmd）
    • 从后台到控制端，称为指令返回（cmdret）

    每种流向都有对应的主题，括号中的是主题名称的前缀。

    MQTT协议设置

    服务质量（QoS）

    绝大部分通信依赖“正好一次”级别的服务质量。即QoS=2。

    消息负载（payload）格式

    使用JSON格式。每个主题都规定了消息负载的格式（由示例即可确定）。请关注示例中字段的类型。

    ID

    如无特殊说明，所有种类的id不限制长度，但只能由字母或数字组成。

    闸机id

    每台闸机都有自己的唯一标识符（machine_id）。只有登记过的闸机才能与后台交互。

    游客id

    游客的id存放在其随身携带的超高频芯片中。只有登记过的游客id才合法。

    指令id

    控制端向后台发送指令时，会包含一个id。后台返回指令结果时，也会附带同样的id。以此来区分结果属于哪条指令。

    MQTT Broker

    以下是连接到MQTT Broker服务器的必要信息。此账号由闸机、后台和控制端共用。

    IP：124.70.155.130
    端口：1883
    用户名：user1
    密码：helloworld

    主题索引

    下面是每个主题的详细说明。红色字是宏，需要用实际值替换（见其说明）。

    主题 说明 发布方 订阅方
    rqst_val_acc_machine_id 链接 闸机 后台
    rply_val_acc_machine_id 链接 后台 闸机
    cmd_add_machine 链接 控制端 后台
    cmd_add_tourist 链接 控制端 后台
    cmd_list_machine 链接 控制端 后台
    cmd_list_tourist 链接 控制端 后台
    cmd_clear_machine 链接 控制端 后台
    cmd_clear_tourist 链接 控制端 后台
    cmdret 链接 后台 控制端

    后台错误代码

    当后台返回非零的错误代码时，代表后台发生了异常。此时，与错误代码一同返回的其他所有值均无效。错误代码的含义如下表所示：

    代码 错误名 含义
    0 无异常 无错误发生，其他值有效。
    1 服务器异常 由于服务器自身原因无法处理请求。
    2 不合法的消息 无法解析提供的JSON消息。

    例如，当后台返回如下信息时，应该视为发生错误，"tourist_id"和"status"的值均无效。

    { "tourist_id": "tEsTtOuRiSt", "status": "granted", "error_code": 1 }
